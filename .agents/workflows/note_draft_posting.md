# Note自動下書き投稿ワークフロー

ユーザーから「Noteの下書きを作って」「Noteにアップして」と指示された際、以下の手順と新しいルールで進めてください。

## 0. 事前確認（Markdownテンプレートの厳格な構造ルール）
対象となる記事のMarkdownファイルを作成・修正する際、以下の「Note下書き用 新ルール」に準拠しているか必ず確認してください。

### 【Note下書き用 新ルール】
1. **メタデータ（上部）**
   - ファイル上部には必ずH1タイトル（`# 🫘 タイトル`等）を入れる。
   - 読了時間（`> ⏱️ **読了時間**: 約X分`）は入れる。
   - ❌ **参照論文数（`> 📄 **参照論文**: X本`）は入れない。**

2. **メタデータ（下部フッター）**
   - ❌ **YAML形式のフッター（`tags:`, `update:`など）は一切残さない（削除する）。**

3. **表や図**
   - 比較表などは標準のMarkdownテーブル（`|---|`）を使用する。
   - ❌ **Mermaidの図などは残さず、全て削除する。**

4. **記事構成**
   - 記事末尾の**「飼い主への説明ガイド」「参照論文」**が抜け落ちていないか必ずチェックする。

▽ 期待されるMarkdownのひな形
```markdown
# 🐕 タイトル
> ⏱️ **読了時間**: 約X分

## 🎯 結論
（テキスト）

## 🗺️ 見出し
| 表のヘッダー1 | 表のヘッダー2 |
|---|---|
| 行 | 行 |

### ▼ 小見出し
（テキスト）

**臨床アクション**
（アクション内容）

## 🗣️ 飼い主への説明ガイド
（Q&Aや説明文など）

## 📚 参照論文
（箇条書き）
```

## 1. 自動投稿スクリプトの準備と実行（--draft 必須）
対象となる記事の「公開予定日（例：2026-04-01）」を `sns_schedule.json` もしくは指示から特定し、以下のコマンドで実行してください。
⚠️ **必ず `--draft` フラグを付与して下書きとして保存し、いきなり公開しないこと。**

```bash
cd c:\Users\souhe\Desktop\VetEvidence_SNS_Drafts\VetEvidence_AutoPost
python auto_post_note.py --date "YYYY-MM-DD" --draft
```

## 2. スクリプトの実行と監視（自動化フロー）
このスクリプトはPlaywrightを使用してブラウザを操作し、以下の処理を自動で行ってくれます。
1. Noteの執筆画面を自動で開く
2. サムネイル画像（note_thumbnail.png）のアップロードとクロップモーダルの突破
3. Markdownから変換されたHTMLのクリップボード経由でのペースト（レイアウト保持）
4. ❌ **タイトルやハッシュタグは（専用の入力枠には）自動入力しない。**
   - 代わりに、Markdown内のH1タイトル（`# 🫘 タイトル`）はサニタイズ時に削除せず、そのまま本文領域にペーストさせる仕組みにします。

※実行後、`コマンドID`のステータスを追跡し、`Status: DONE` になるまで待機してください（通常約30秒〜1分）。

## 3. 結果の報告
実行完了後、Noteの管理画面に記事が正しく下書き生成された旨をユーザーに報告してください。
