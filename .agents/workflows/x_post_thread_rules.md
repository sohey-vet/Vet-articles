# X自動スレッド投稿（Playwright）の整形・文字数管理ルール

このドキュメントは、Xへの無制限なテキスト投稿によるインプレッション低下や文字数オーバーを防ぐための、最も根幹となる「文章整形・2段階ツリー化ロジック」の設計仕様書です。

## 1. 投稿文章の構造と分離基準

自動投稿では、元データ（`sns_schedule.json`）のひと繋がりの文章を、そのまま投稿してはいけません。
以下のルールに従い、投稿プログラム（`auto_post_x.py`）内で動的にテキストを分離・整形してください。

### 【ルール1】ハッシュタグの完全削除
文章内に含まれる全てのハッシュタグ（例：`#獣医 #動物救急 #急性下痢` など行末にあるもの）は、**必ず正規表現（`re.sub(r'#\S+', '', text)`等）を用いて完全に除外**します。
これは、無駄な文字数を削り、外部リンク誘導時のアルゴリズム制約を回避するためです。

### 【ルール2】「根拠」部分の完全削除と、外部リンク・Note誘導ブロックの強制リプライ化
Xの文字数節約およびスッキリとした見た目のため、「**`📄 根拠:`**」から始まる文章は1枚目・2枚目どちらにも含めず、**完全に削除（破棄）**します。

さらに、Xのアルゴリズム上**「外部リンクを含むツイートはインプレッションが大きく低下する」**という仕様があるため、総文字数が制限（例：500文字）以内であっても、以下のルールでテキストを動的に切り離します。

#### ▼ 1ツイート目（メインポスト）
* **内容:** 最初の見出し行 ＋ 記事の要約部分から、末尾の誘導・リンク部分を取り除いた文章。
* **分離の目安:** 「～日々尽力しているのです。」など、記事の結論部分まで。
* **画像:** 記事関連のサムネイル画像（`.png`）をここにのみ添付する。
* **制限事項:** 「根拠」以降の文字列やURL、Noteへの誘導フレーズ、ハッシュタグは一切含めない。

#### ▼ 2ツイート目（自己リプライ / スレッド）
* **内容:** 外部URL（`note.com` 等）を含む段落ブロックと、その直前にあるNote誘導文（例: `詳細な実践法はプロフのNoteリンクにて。` や `💡 疾患の詳細は～` など）**のみ**を束ねたもの。
* **画像:** 無し（OGPプレビュー等のカードは表示された瞬間に即座に削除する）。
* **目的:** アルゴリズムの評価低下を防ぎつつ、外部プラットフォーム（Note等）への出口をここにカプセル化する。

## 2. 処理フロー実装の注意点

* **分離文字列の判定（動的切り出し）:** テキストを空行（パラグラフ）ごとに調査し、`http:` や `note.com` 等のリンクが含まれるブロックを見つけた場合、そこを「ツリー（2ツイート目）の終端」と見なすこと。さらにその直前のブロックが「短めの文字数」または「詳細・ Note・💡・🔗」等を含む誘導文であれば、それも遡ってツリーに含めること。
* **分離時の空行処理:** `.strip()` などを駆使し、分離後のメインポストとリプライの両方で不要な改行や空白が先頭・末尾に残らないよう整形すること。
* **スレッド作成のUI担保:** 画像が添付されたことでUIの「＋（ポストを追加）」ボタンが隠れる症状を防ぐため、JSでの `scrollIntoView()` ＋ ネイティブクリックなどの堅牢化ロジックを使用すること。また、生成されたテキストボックスの総数（`len(textboxes)`）を配列インデックスで確認し、意図せず1枚目に追記されないようガードを厚くする。
