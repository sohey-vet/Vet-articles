## 🚨 絶対に守るルール

### 1. 結論（🎯 結論）のフォーマット
- MDの結論ヘッダーは必ず **`## 🎯 結論`** とする。
- `## 結論（30秒で読める）` や `## 結論 (Take-home Message)` は **禁止**。変換スクリプトが認識できず空の結論が出力される。
- 結論は **箇条書き（`-` リスト）4〜5項目** で書く。段落形式は禁止。
- 各項目は **1文で簡潔に**。太字`**...**`でキーワードを強調。
- **「Take-home Message」というテキストは一切使用禁止。**

### 2. 詳細解説セクションの構造
- `## 📖 詳細解説` という空のH2見出しを作ってはならない。
- 詳細解説の各サブセクションはそれぞれ **独立したH2** (`##`) として書く。
- H3 (`###`) で書くと変換スクリプトが空のアコーディオンを生成してしまう。
- 例: `## 1️⃣ セクション名`、`## 2️⃣ セクション名` のように番号付きH2で書く。

### 3. 編集情報・Note投稿チェックリストの禁止
- MD末尾に `## ✏️ 編集情報` を書いてはならない。HTMLに変換されてしまう。
- `<!-- Note投稿時のチェックリスト -->` のコメントも入れない。
- メタデータブロック（``` ジャンル: ... ```）も入れない。

### 4. index.html の data-tags ルール
- `data-tags` には必ず**サイドバーのジャンルフィルター名**を1つ以上含める。
- 有効なジャンル名: `救急`, `循環器`, `輸液`, `腎泌尿器`, `猫`, `皮膚`, `免疫`, `血液`, `消化器・肝臓`, `腫瘍`, `神経`, `内分泌`, `麻酔`, `眼科`, `歯科`, `その他`
- 「抗菌薬の適正使用」のような**サイドバーに存在しないタグをdata-tagsに入れてはならない**。検索できなくなる。

### 5. 変換スクリプト実行時の注意
- `python scripts/md_to_site_html.py --file <path>` で個別変換。
- Windows環境では `$env:PYTHONIOENCODING="utf-8"` を設定してから実行。
- 変換後は必ずHTMLの以下を確認:
  - 結論が箇条書きで正しく出力されているか
  - `<li>--</li>` のようなゴミがないか
  - `</em>*` のような壊れた変換がないか
  - 詳細解説のアコーディオンが空でないか

### 6. 記事の重複防止
- 同じ内容の記事が異なるファイル名で存在してはならない。
- 新記事作成前に `topics/` 配下の既存ファイルを必ず確認する。
- 例: `手作り食_生食のリスクとエビデンス.html` と `手作り食と生食のリスクとエビデンス.html` の重複が発生した。

### 発見されたバグの根本原因
1. **結論が空になる**: MDのヘッダーが `## 結論（30秒で読める）` だと変換スクリプトが認識しない
2. **詳細解説が空になる**: `## 📖 詳細解説` の直下にテキストがなく `### サブセクション` のみだと空アコーディオンが生成される
3. **`<li>--</li>` ゴミ**: MDの `---`（区切り線）が箇条書き直後にあると `<li>--</li>` に変換される
4. **`</em>*` ゴミ**: MDの `**太字**` が段落の最後にあると `<em>内容</em>*` に誤変換される
5. **Take-home Message残存**: 古い変換ロジックで生成されたHTMLに残っていた
