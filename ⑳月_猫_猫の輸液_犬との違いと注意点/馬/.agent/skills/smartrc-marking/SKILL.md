## 1. 印付けルール（最重要）

ユーザーから3つのリスト（A, B, C）を受け取り、馬がどのリストに属するかで印と列が決まる。

| 条件 | 列 | フィールド | 印 | 備考 |
|------|-----|-----------|-----|------|
| A∩B∩C | 主 | `m1` | `注` | 3リスト全てに該当 |
| A∩B（Cなし） | 主 | `m1` | `◎` | AとBの両方に該当 |
| A∩C（Bなし） | **主＋仮** | `m1` + `m2` | `○` + `○` | **2行設定する** |
| B∩C（Aなし） | **主＋仮** | `m1` + `m2` | `○` + `◎` | **2行設定する** |
| A単独 | 仮 | `m2` | `○` | Aのみに該当 |
| B単独 | 仮 | `m2` | `◎` | Bのみに該当 |
| C単独 | 仮 | `m2` | `▲` | Cのみに該当 |

> [!CAUTION]
> **A∩CとB∩Cは1頭に対して2行（m1とm2の両方）に印を付ける。**
> これを忘れて1行だけ設定するミスが過去に発生している。
> `compute_marks()` 関数の該当箇所を参照:
> ```python
> elif "A" in s and "C" in s:
>     result[venue][rno].append((uno, "m1", "○", hname))
>     result[venue][rno].append((uno, "m2", "○", hname))
> elif "B" in s and "C" in s:
>     result[venue][rno].append((uno, "m1", "○", hname))
>     result[venue][rno].append((uno, "m2", "◎", hname))
> ```

### ルール適用の優先順位

判定は上から順に行い、**最初にマッチしたルールのみ適用**する（A∩C/B∩Cの2行設定を除く）:

1. A∩B∩C → `m1: 注`
2. A∩B → `m1: ◎`
3. A∩C → `m1: ○` + `m2: ○`
4. B∩C → `m1: ○` + `m2: ◎`
5. A単独 → `m2: ○`
6. B単独 → `m2: ◎`
7. C単独 → `m2: ▲`

## 2. データ入力フォーマット

### LIST_A / LIST_B の形式
```python
LIST_A = {
    '東京1R': [(3, 'ピコチャンブルー'), (4, 'セイウンマカロン')],
    '阪神3R': [(1, 'テーオーシュテルン'), (2, 'キングスペード')],
}
```

- キー: `'会場名+レース番号R'`（例: `'東京1R'`, `'阪神12R'`）
- 値: `(馬番, '馬名')` のタプルのリスト

### LIST_C の形式
```python
LIST_C = {
    '東京1R': [(9, 'アイズ')],     # ← 「R」付き
    '小倉9': [(8, 'ゴーゴータカシ')],  # ← 「R」なしでもOK
}
```

- LIST_Cのキーは `R` が付かないことがある（ユーザーの元データの形式による）
- `normalize_race_key()` が `'東京1R'` も `'東京1'` も `('東京', 1)` に正規化するので問題ない

### parse_input.py の使い方

ユーザーが生テキスト（コピペ形式）でデータを渡してきた場合は `parse_input.py` で変換できる:
1. `raw_data` 変数にテキストを貼り付ける
2. `python parse_input.py` を実行
3. 出力される `LIST_A = {...}`, `LIST_B = {...}`, `LIST_C = {...}` を `smartrc_auto.py` にコピー

`東京`, `中山`, `京都`, `阪神`, `小倉`, `新潟`, `福島`, `中京`, `札幌`, `函館`

## 3. スクリプト技術仕様

### 本番スクリプト: `smartrc_auto_v12.py` (v12)

**特徴:**
- 会場自動検出（ホーム画面の当日の日付を含む行からボタンを自動識別）
- `VENUES` 変数の手動設定は **不要**
- UIクリック方式で印を設定（Store API直接操作ではない）
- 自動ログイン機能あり

**ナビゲーション戦略:**
- 各会場の最初のレース: ホーム → 会場ボタンクリック → レース一覧 → 該当Rクリック
- 同会場の2レース目以降: 画面上部の「↓（次のレースへ）」矢印ボタンをクリックして、目的の対象レースへ超高速ジャンプ。
  - **超重要（SPA/Ext JSの罠）**: SmartRcは過去の画面（一覧画面など）を非表示(`display:none`にならないこともある)でDOMツリーに大量に残すSPA構造を持つ。そのため、`document.querySelectorAll('.x-button')` などで単純にボタンを探すと、**裏側に隠れている別の画面のボタン**を誤検知して押してしまい、レース一覧にフォールバックしてしまうバグが発生する。
  - **解決策 (`elementFromPoint`)**: ↓ボタン（次のレース）を探す際は、必ず `document.elementFromPoint(centerX, centerY)` を使って「ユーザーの目に確実に見えている要素」だけをフィルタリングすること。
  - さらに、クラス名は `fa-angle-down`（ドロップダウン等）ではなく、**`fa-arrow-down`** を持つ要素を厳密にターゲットにすること。

**印設定の仕組み（`set_marks`関数）:**
1. 馬名テキストを含む`<tr>`内の`mark`クラスを持つ`<td>`を見つける
2. `scrollIntoView` してクリック（印選択UIが出現）
3. `.x-button-flat` の中から目的の印テキストを探してクリック
4. `m1`(主)なら1番目の該当ボタン、`m2`(仮)なら2番目の該当ボタンをクリック
5. 「確認」や「OK」ダイアログが出たら閉じる

## 4. 絶対に守る技術的制約

### 🔴 Chrome関連
- **`taskkill /F /IM chrome.exe` は絶対禁止** — ユーザーの全Chromeが死ぬ
- 専用プロファイル `robot_chrome_profile` + デバッグポート `9222` で分離
- ロックエラー時はタスクマネージャーで該当プロセスだけ手動終了

### 🔴 ログイン確認
- `Ext.isReady` は使わない
- 「ログアウト」ボタン（`.x-button` で `innerText === 'ログアウト'`）の可視性で判定

### 🔴 印の操作方法
- **UIクリックで操作する**（Selenium `ActionChains` → `move_to_element` → `click`）
- **Ext JS Store API (`rec.set('m1', '◎')`) は使わない** — 値は変わるがサーバーに保存されない
- `.x-button-flat` 内の印ボタンを `text.strip() == mark` で照合

### 🔴 要素操作
- クリック前に必ず `scrollIntoView({block:'center'})` を実行
- クリック後は `time.sleep()` で遷移を待つ（最低0.3秒、ページ遷移は3〜5秒）
