# SNS自動投稿ロジック（Threads編）必須スキル・メモリ

このドキュメントは、将来のSNS（特にThreads）の自動投稿ロジックやドラフト作成において**絶対に順守すべき運用ルール**を定義したものです。

## 1. リンク（外部URL）の完全排除
Threadsのアルゴリズムにおいて、外部URL（`note.com`など）を投稿に含めるとインプレッションが極端に低下する（シャドウバン状態になる）仕様が確認されています。
*   **絶対ルール**: Threadsへの投稿テキスト（メイン本文、およびツリーのリプライ）には、**いかなる外部URL文字列も絶対に含めてはいけません。**
*   **リンクカード（OGP画像）の抑止**: URLを排除することで、巨大なリンクプレビュー（顔写真などのOGP）が自動生成される仕様自体を無効化し、見栄えの悪化を防ぎます。

## 2. メイン投稿（1枚目）の構成要素
*   **画像添付**: 一枚目のメイン投稿には必ずサムネイル画像を添付して視覚的フックを作ります。（※`auto_post_threads.py`内では`DataTransfer`を用いたPasteイベントで強制添付する処理を採用）
*   **ダサい2重ツリーの防止**: 本文の最後に不自然に「詳細はこちら👇」のような短いCTA（Call to Action）を追加しては**いけません**。Threadsの文字数分割ロジック（約480文字）によって、CTA部分だけがはぐれて独立した「不要な2枚目の投稿」になってしまうことを防ぎます。

## 3. ツリー（リプライ枠・2枚目）のプロフ誘導文（厳格指定4パターン）
外部URLの代わりに、アカウントのプロフィールへの誘導を行います。プロフリンクを踏ませるためのメンション（`@pawmedical_jp`など）は**記載不要**（文字のみで直接タップを促す仕様に最適化済み）です。
以下の4パターンを、記事のキーワード内容に合わせて**必ず**使い分け・自動生成するようにスクリプト（`auto_post_threads.py`）を構成してあります。

1.  **［論文・ガイドライン系］**
    （検知ワード：論文, リファレンス, 文献, ガイドライン, aaha, wsava, jvim）
    `📚 今回のトピックの完全版（論文リファレンス付き）は、プロフィールのURLからNote記事を参照してください`

2.  **［救急・治療・薬系］**
    （検知ワード：薬, 投与, 用量, 救急, ショック, 治療, 麻酔, 初期対応）
    `🏥 診療で使えるガイドラインや推奨薬の詳細は、プロフのリンク先にある記事の中でさらに深堀りしています`

3.  **［病態・生理系］**
    （検知ワード：病態, 生理, 機序, メカニズム, ホルモン, 鑑別, 原因）
    `📝 「さらに掘り下げた細かい病態生理や鑑別も知りたい！」という方は、プロフィールにあるリンクからNote全編をぜひご覧ください`

4.  **［汎用（いずれも該当しない場合）］**
    `💡 疾患の詳細や、明日から使える最新エビデンスは、プロフィール欄のNoteリンクから解説記事をチェックしてみてください🐾`

---
※ 今後、投稿スクリプトや記事の生成ロジックを改修する際は、必ずこの**「URL排除ルール」**および**「2重ツリー回避のCTA削除」**、**「4指定パターンの正確な適用」**の原則を維持してください。

## 4. 火・木・土（引用投稿）の運用ルール
火・木・土は、前日（月・水・金）のメイン記事を「引用ポスト（Quote）」形式で再掲する日です。この際の挙動について、以下の点を**絶対的にお忘れなきよう**ご注意ください（過去にAIが誤ってロジックを改変し、テキストがスカスカになる事故が発生しました）。

*   **専用テキストの維持**: スケジュールファイル（`sns_schedule.json`）の「Threads Short」等に記載されている**火木土専用のしっかりとした長文テキストをそのままメイン本文として使用**してください。プログラム側で、その場の思いつきのような短いランダム文（例:`generate_quote_text`等）で上書きして内容を薄くしてはいけません。
*   **プロフ誘導文のツリー追加**: 引用投稿であっても、通常の投稿と同様に**ツリー（リプライ）部分に上記の【厳格指定4パターン】のプロフ誘導文を自動判定してぶら下げます**。`reply_text = ""` と置いてツリーを消去してはいけません。
*   **画像の添付不要**: 引用ポストでは元投稿がプレビューカードとして展開されるため、別途画像を添付する必要はありません（`image_path = None`）。
