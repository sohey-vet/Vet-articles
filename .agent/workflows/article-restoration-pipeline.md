# 記事復旧とNote原稿生成の完全ワークフロー (article-restoration-pipeline)

このワークフローは、3/20の大量コンパイル障害によって中身が空になった（メタデータのみになった）残りの58本のMarkdown記事を、視覚的バグや構文エラーを一切起こさずに「完全なGold Standard状態」へと復元し、同時にNoteの完璧な下書きを再生成するための厳密な手順書です。

## フェーズ 1: HTMLの安全なサルベージと構文修復
1. **Gitからの強制引き出し**: 
   `git checkout b1ba6afb^ -- <filepath.html>` などを実行し、破損が起きる前の「表示が完璧だった時代のHTML」を直接リポジトリから取り出します。
2. **Mermaid 10.9.1 構文エラーの完全除去 (超重要)**:
   過去のHTMLに埋め込まれている生のMermaidコードブロック内に「**重複したノード定義（例: 末尾での `N["安定化後..."]` の再宣言）**」が存在する場合、現在のMermaidエンジン(v10.9.1)では致命的なパースエラーを引き起こします。
   * **症状**: 図解が「Syntax error in text」となり表示されないだけでなく、**ページ全体のJavaScriptが連鎖的にクラッシュするため、「参照論文のアコーディオン」が一切開かなくなる**という重大なバグを併発します。
   * **対策**: HTMLファイル内の `<pre class="mermaid">` ブロックを必ず目視またはスクリプトで検査し、矢印で結ばれていない単独の重複ノード定義を削除してください。

## フェーズ 2: Markdownファイルの黄金再構築 (Reverse Transpilation)
1. **Pythonスクリプトによる逆変換**:
   修復したHTMLを元に `mass_restore_to_gold_md.py` 等の逆変換スクリプトを走らせ、ベースとなるMarkdownファイルを生成します。
2. **参照論文の手動（または専用スクリプトによる）注入**:
   既存の逆変換アルゴリズムは複雑なアコーディオン構造（`<div id="refs">` 等）をスキップするため、Markdown側に「参照論文」が抽出されません。
   必ず復元後の `.md` ファイルの末尾（`---` の上）に、HTMLから抽出したリストを `## 📚 参照論文` の形式で**追記補完**してください。これを忘れると、次段のNote投稿時に参考文献が欠落します。
3. **Mermaid記法の同期**:
   HTML側で修正したMermaidの構造と一言一句違わない形で、Markdown側にもコードブロックを配置してください。（※現状、`<div class="mermaid-wrapper"><pre class="mermaid">` という生のHTMLタグでMarkdown上に埋め込む方針が安定しています）

## フェーズ 3: 自動化Noteスクリプトへの連携保証
`auto_post_note.py` を実行してNote下書きを生成する前に、スクリプトのフィルタリング・ロジックが以下の基準を満たしていることを確認します（※2026-03-24に修正済みですが、念のため確認）。

1. **参照論文の保持**: `junk` 配列の中に `'参照論文'` が含まれていないこと。（含まれているとMarkdownの参照論文ブロックが丸ごと削除されます）
2. **フローチャートの確実な黒塗り（削除）**: Noteには図解テキストを出力してはならないため、`line_s.startswith('```mermaid')` だけでなく、`line_s.startswith('<div class="mermaid-wrapper">')` も検知して、`</div>` が来るまですべての行を `continue` して出力から抹消するフィルターを設けてください。

## フェーズ 4: デプロイと連携
1. `.md` と `.html` を対にして `git add` し、リポジトリへコミット＆プッシュします。
2. その後、`python auto_post_note.py --date <公開日> --draft` を実行してNote下書きを自動生成します。（※図解が削られ、太字装飾が適用され、最下部に参照論文が載った完璧な原稿ができあがるはずです）
