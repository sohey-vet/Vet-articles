# 🧠 Note自動投稿プログラムの固定メモリー (2026/03/23 確立)

Note.comへの「サムネイル付き」での完全自動投稿を安定させるために確立した、**絶対に改変・削除してはならないコア・ロジック**のメモリーです。今後の記事作成プログラムや、仕様変更時のメンテナンスにおいて、未来のAIエージェントおよびユーザーは必ずこのドキュメントを参照してください。

## 1. 見出し画像（サムネイル）アップロードの必須プロセス
Note.comのUIは高度なReact SPAであり、単純な `<input type="file">` への流し込みでは内部状態が更新されずエラーになります。以下の**3ステップのネイティブ入力**を厳守すること。

1. **不可視ボタンのクリック:**
   ヘッダー画像設定用のグレーの丸アイコンはテキストを持っていません。
   ✅ 正解ロジック: `page.locator('button[aria-label="画像を追加"]').first`
2. **OSネイティブダイアログの迎撃:**
   Playwrightの `expect_file_chooser()` を用いてOSのファイルダイアログを直接フックし、`file_chooser.set_files(thumb_path)` でローカル画像を流し込む。
3. **[最重要トラップ] クロップモーダルの突破:**
   画像アップロード直後、Note.comは画面中央に「クロップ(切り抜き)モーダル」を強制出現させます。これを放置して投稿へ進むと**画像が破棄されてサムネ無しで公開されます**。
   ✅ 正解ロジック:
   ```python
   # モーダル右下の「保存」を完全一致で探し出し、強制クリックする
   save_modal_btn = page.locator('button:has-text("保存")').get_by_text("保存", exact=True).first
   save_modal_btn.click(force=True)
   ```

## 2. 本文・タイトルの入力ロジック
- **タイトル入力:** オーバーレイ等による干渉を防ぐため、物理的な `click()` ではなく仮想イベント付きの `fill()` を使用すること。
  `title_box.fill(title, timeout=15000)`
- **HTMLのクリップボードペースト:** 
  太字(`<strong>`)や表組みを適切にレンダリングさせるため、文字列の直接入力ではなく、`ClipboardItem` APIを用いて `text/html` フォーマットでクリップボードにコピーした上で、Playwright経由で `Control+V` (Mac: `Meta+V`) を送信すること。

## 3. 最終公開（Publish）ボタンのロジック
Note.comの右側にスライドしてくる「公開設定」パネル内の最終クリックボタンは、アップデートにより内部ノードのテキスト（「公開」「投稿」「今すぐ公開」など）が変動しやすくなっています。
- 正規表現での完全一致 `^公開する$` は失敗する確率が高いため禁止。
- ✅ 正解ロジック: 許容度の高い `has-text` を用い、一番下のボタンをターゲットにする。
  `final_pub_btn = page.locator('button:has-text("公開"), button:has-text("投稿")').last`
  `final_pub_btn.click(force=True, timeout=15000)`
